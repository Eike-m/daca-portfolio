# Nädal 0 — Onboarding

## Projekti eesmärk

Nädal 0 eesmärk oli seadistada kõik peamised tööriistad ja keskkonnad, mida kasutame DACA programmi jooksul andmeanalüüsi projektide loomiseks. Lisaks individuaalsetele seadistustele õppisime kasutama ühiseid meeskonnatöö tööriistu ning lõime esimese „Hello, UrbanStyle!“ vertikaalse töövoo. 

## Peamised tegevused

- Seadistasime **Supabase** andmebaasi ja õppisime SQL päringute kasutamist.
- Lõime **GitHubi** konto ja `daca-portfolio` repositooriumi versioonihalduseks ja portfoolio ehitamiseks.
- Seadistasime **VS Code + Python** arenduskeskkonna.
- Installeerisime **Power BI** või Python visualiseerimise tööriistad.
- Kasutasime **NotebookLM-i**, laadisime üles RAG failid ning genereerisime AI Audio Overview kokkuvõtteid.
- Õppisime, kuidas ühendada:
  - andmed (Supabase),
  - kood (VS Code),
  - versioonihaldus (GitHub),
  - ja AI õppekeskkond (NotebookLM). :contentReference[oaicite:2]{index=2}

## Grupitöö — Tooteanalüüsi osakond

Sessioon 2 grupitöö käigus lõime meeskonnaga ühise GitHub repository, Supabase projekti, NotebookLM notebook’i ning Team Charter’i ehk meeskonna kokkulepped. Grupitöö eesmärk oli õppida professionaalset koostööd, rollide jagamist, dokumenteerimist ning ühiste süsteemide seadistamist.

Osalesin Tooteanalüüsi osakonna töös, kus meie ülesandeks oli luua ühine andme- ja õppekeskkond. Tutvusime omavahel, saime ülevaate DACA programmi 11-nädalasest teekonnast ning UrbanStyle.ltd ettevõttest, mille andmeid programmi jooksul analüüsime.

Lisaks õppisime AI baasteadmisi ja tutvusime NotebookLM-iga kui AI-õppekaaslasega, mis aitab õppematerjale analüüsida ja audioülevaateid genereerida. Mõistsime, et AI kasutamine on oluline osa kaasaegse andmeanalüütiku töövoost.

Grupitöö käigus seadistasime:
- GitHub keskkonna versioonihalduseks,
- Supabase andmebaasi,
- NotebookLM AI-õppekeskkonna,
- ning viisime läbi esimese demo-esitluse, kus näitasime töötavaid süsteeme ja meeskonna koostööd.

## Peamised õppetunnid

- Andmebaas lahendab Exceli versioonikaose ja võimaldab töötada ühe tõeallikaga.
- GitHub ei ole ainult failide hoidla, vaid professionaalne portfoolio ja versioonihalduse tööriist.
- VS Code ühendab SQL-i, Pythoni ja GitHubi üheks töövooguks.
- NotebookLM aitab õppematerjale kiiremini mõista ja küsimustele vastata AI abil.
- Vertikaalne töövoog tähendab, et andmed, kood ja tulemused töötavad tervikuna koos. 

## AI kasutamine

Kasutasin AI abi seadistusvigade lahendamisel, SQL süntaksi mõistmisel ning GitHubi ja VS Code’i töövoogude seadistamisel. NotebookLM aitas õppematerjalidest kiiremini aru saada ning selgitas UrbanStyle’i ärikonteksti ja DACA programmi ülesehitust.
